Berechnung statistischer Bildgrößen

In dieser Übung soll eine Anwendung erstellt werden, die in der Lage ist, ein Bild anzuzeigen und seine wichtigsten statistischen Kenndaten zu berechnen. Mit der Bearbeitung dieser Aufgabe lernen Sie darüber hinaus, wie man den Java AWT-Grafikkontext benutzt, um eigene Zeichnungen zur Anzeige zu bringen.

Allgemeine Hinweise

  • Diese Aufgabe kann in Gruppen zu maximal 2 Personen bearbeitet werden.
  • Ein Teil der Aufgabe ist optional.

Aufgabenstellung

  1. Nehmen Sie dieses Archiv als Ausgangsbasis für Ihre eigene Arbeit und bringen Sie das Programm unter Eclipse zum Laufen. Benennen Sie das Projekt gemäß der Konvention für die Abgabe Ihrer Übungen um. Analysieren Sie das Programm und machen Sie sich mit der Funktion der einzelnen Klassen vertraut. Betrachten Sie vor allem die mit "TODO" gekennzeichneten Stellen.
    • ImageAnalysis.java: Darin befindet sich das Hauptprogramm mit dem Layout des GUI.
    • HistoView,java: Eine von java.swing.JPanel abgeleitete Klasse, die zur Anzeige eines Histogramms dienen soll.
    • StatsView.java: Eine von java.swing.JPanel abgeleitete Klasse, die zur Anzeige statistischer Bildgrößen dienen soll.
    • ImageView.java: Hilfsklasse zur Anzeige von Bildern wie aus den letzten Übungen.
       
  2. Legen Sie in ImageAnalysis ein Array an, das Sie mit den ARGB-Pixeln des geladenen Bildes füllen, bevor dieses Bild irgendwie verändert wird ("Original-Pixel"). Dies wird der Ausgangspunkt all Ihrer Bildmanipulationen (Helligkeit, Kontrast, Autokontrast siehe Punkt 5 und 6). An welchen Stellen im Source-Code müssen Sie das Array der "Original-Pixel" befüllen?
     
  3. Berechnen und zeichnen Sie ein Histogramm der Graustufen des aktuell angezeigten Bildes.
    • Berechnen Sie in processImage() der Klasse ImageAnalysis das Histogramm. Übergeben Sie das Resultat der Klasse HistoView und der Klasse StatsView.
    • Implementieren Sie in HistoView das Zeichnen des Histogramms. In der Vorgabe sehen Sie, wie man Linien zeichnet. Nun müssen Sie für jeden Eintrag im Histogramm-Array eine geeignete senkrechte Linie zeichnen. Das Histogramm soll so skaliert werden, dass der höchste Peak die volle Höhe des Anzeigebereichs ausnutzt.
       
  4. Berechnen Sie die folgenden statistischen Signalgrößen aus dem Histogramm, das Sie zuvor der Klasse StatsView übergeben haben:
    • Minimalwert
    • Maximalwert
    • Mittelwert
    • Varianz
    • Medianwert
    • Entropie [Bits]
    Zeigen Sie die berechneten Werte in StatsView an. Hier ein Screenshot.

    Hinweis:     Die Folien aus der Vorlesung beinhalten Berechnungsvorschriften für diese Größen auf Grundlage eines Histogramms. Einzige Ausnahme ist der Medianwert. Falls Sie den Median wie auf den Folien angegeben direkt aus den Graustufen des Bildes berechnen wollen, dann müssen Sie der Klasse StatsView zusätzlich das aktuelle Pixel-Array übergeben. Alternativ können Sie Median auch aus dem Histogramm berechnen wollen.
      
  5. Jetzt sollen im Programm Funktionalitäten zur Veränderung des Bildes eingebaut werden. Verwenden Sie den vorhandenen Helligkeitsregler, um die Helligkeit des Bildes zu erhöhen oder zu erniedrigen. Ergänzen Sie das Programm um einen weiteren Slider, der den Kontrast des Bildes verändert. (Was ist ein geeigneter Wertebereich für den Kontrast-Slider?) Achten Sie darauf, dass kein Over- oder Underflow auftritt. Die Graustufe eines Pixels muss immer im Bereich zwischen 0 und 255 liegen.
     
  6. Optional: Als optionale Zugabe kann die Funktion "Auto-Kontrast" realisiert werden. Fügen Sie dafür einen entsprechenden Button hinzu. Im einfachsten Fall nehmen Sie für den Histogrammausgleich die oben berechneten Minimal- und Maximalwerte der Graustufen. Oder Sie implementieren den "Auto-Kontrast" wie in gängigen Bildverarbeitungsprogrammen: Dazu bestimmen Sie im Histogramm die Graustufen, bis zu denen 1% bzw. 99% aller Werte liegen. Diese Werte verwenden Sie dann anstelle des Minimums bzw. Maximums, um den Histogrammausgleich des Eingangsbildes durchzuführen.
     
    Führen Sie die Transformation der Helligkeitswerte entsprechend dem nebenstehenden Bild durch und geben Sie das neue transformiere Bild aus. Ein geeignetes Testbild mit sehr geringer Dynamik finden Sie hier.
